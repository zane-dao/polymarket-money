# 2026-07-22 17:58 Asia/Singapore｜市场回放布局与策略竞技场交互修复

## 目标

修复市场回放事件流显示不全、当前后端事件占用过多空间，以及策略竞技场比较所选运行无可见反馈；
范围仅限本地 paper 工作台，不启动联网采集或真实交易。

## 事实与证据

- 原回放布局把当前事件放在宽主栏、100 条事件流放在 380px 侧栏，且两者没有有界滚动区。
- 原竞技场只在状态文字中更新比较结果，没有进行态，后端拒绝时也没有醒目的错误区域。
- 前端 Vitest `24/24`、全仓 TypeScript、Vite production build 与差异格式检查通过。
- 全仓 Node 在受限沙箱中除 loopback 监听外 `60/60` 通过；沙箱外 Web 专项 `7/7` 通过。

## 修改

- AI 项目层：更新当前验证基线、Batch 07 前端报告和本摘要索引。
- 代码层：回放事件流改为宽主栏有界滚动表格，详情改为窄栏检查器；竞技场增加选择计数、忙碌态、
  成功态、错误态和结果表滚动保护，并新增后端拒绝的组件测试。后续将零成交回测的 `winRate=null`
  统一为合法的“不可用”指标，修复比较 DTO 解析并增加合同测试。
- 外部状态：无变化；没有启动公网采集、常驻进程或真实账户操作。
- 根因修复：后端策略合同新增有效默认值，界面不再使用参数下限；L V2 回测恢复冻结候选配置；
  版本号自动递增，加载旧版本不会覆盖下一次保存目标。
- 公平比较：新回测默认验证集并持久化数据分组、时间跨度、执行场景、样本指纹和样本数；竞技场
  拒绝缺少范围证据或样本范围不一致的运行。
